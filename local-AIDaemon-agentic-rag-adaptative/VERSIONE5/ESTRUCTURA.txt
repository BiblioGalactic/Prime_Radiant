ARCHIVOS Y RESPONSABILIDADES

┌─ config.py
│  Responsabilidad: Detectar rutas y cargar configuración
│  
│  Funciones principales:
│  ├─ get_base_path()       → Directorio raíz del proyecto
│  ├─ get_model_path()      → Ruta del modelo GGUF
│  ├─ get_index_path()      → Ruta del índice FAISS
│  ├─ get_embeddings_model() → Nombre del modelo de embeddings
│  ├─ get_context_window()  → Fragmentos a recuperar (default: 5)
│  ├─ get_max_tokens()      → Tokens máximos (default: 512)
│  └─ get_temperature()     → Temperatura (default: 0.7)
│
│  Importaciones: pathlib, os (solo stdlib)
│  Líneas: 78
│  Fallos: Rápido y claro si faltan archivos
│
└─ Uso: from config import MODEL_PATH, INDEX_PATH, ...

┌─ llm.py
│  Responsabilidad: Interfaz con modelo LLM via llama-cli
│
│  Clase: SimpleLLM
│  ├─ __init__()              → Verifica llama-cli disponible
│  ├─ query(prompt) → str     → Envía prompt, retorna respuesta
│  ├─ stream_query(prompt)    → Generador de tokens
│  └─ _verify_llama_cli()     → Validación interna
│
│  Importaciones: subprocess, config, pathlib
│  Líneas: 99
│  Subprocess: Solo sync (no daemons)
│  Error handling: TimeoutExpired + mensajes claros
│
└─ Uso: llm = SimpleLLM(); response = llm.query("¿Quién eres?")

┌─ rag.py
│  Responsabilidad: Recuperación de contexto con FAISS
│
│  Clase: SimpleRAG
│  ├─ __init__()                    → Carga índice + chunks
│  ├─ search(query) → str           → Retorna contexto
│  ├─ search_with_scores(query)     → Retorna (fragment, score)
│  ├─ _load_index()                 → Lee index.faiss
│  └─ _load_chunks()                → Lee chunks.pkl
│
│  Importaciones: faiss, pickle, numpy, sentence-transformers
│  Líneas: 108
│  Embedding: sentence-transformers/all-MiniLM-L6-v2
│  K: CONTEXT_WINDOW (default 5)
│
└─ Uso: rag = SimpleRAG(); context = rag.search("¿Qué es Python?")

   - sys (stdlib)

   - llm (módulo local)

   - Imprime: "✓ Modelo GGUF: ..."

   - pickle (stdlib)

   - SimpleLLM() → Verifica llama-cli
   - SimpleRAG() → Carga index.faiss + chunks.pkl

   - Imprime: "> Pregunta: "
   - Lee input
   - Ejecuta: rag.search(question)
   - Ejecuta: llm.query(prompt)
   - Imprime respuesta

================================================================================
FLUJO DE UNA PREGUNTA
================================================================================

Usuario: "¿Qué es Python?"
         │
         ├─ main.py: input() → "¿Qué es Python?"
         │
         ├─ WikiRAGE5.answer(question)
         │  │
         │  ├─ rag.search("¿Qué es Python?")
         │  │  ├─ SentenceTransformer.encode() → vector
         │  │  ├─ faiss.search() → índices top-5
         │  │  ├─ Lee chunks[idx] para cada resultado
         │  │  └─ Concatena → contexto (400-600 palabras)
         │  │
         │  ├─ Construye prompt:
         │  │  "Basado en el siguiente contexto, responde...
         │  │   CONTEXTO: [contexto de RAG]
         │  │   PREGUNTA: ¿Qué es Python?"
         │  │
         │  ├─ llm.query(prompt)
         │  │  ├─ subprocess.run(['llama-cli', '-m', model, '-p', prompt, ...])
         │  │  ├─ Espera respuesta (timeout 120s)
         │  │  └─ Retorna stdout
         │  │
         │  └─ Retorna (respuesta, contexto)
         │
         └─ Imprime respuesta
            Opcionalmente: Imprime contexto

GESTIÓN DE ERRORES
================================================================================

Punto            Error probable         Acción
─────────────────────────────────────────────────────────────
config.py        Model no existe        FileNotFoundError + msg claro

Todos los errores tienen mensajes informativos.
Nada falla silenciosamente.

PATRONES UTILIZADOS
================================================================================

1. Singleton pattern (implícito)
   - SimpleLLM: Una instancia en main
   - SimpleRAG: Una instancia en main

2. Dependency Injection (explícito)
   - config.py proporciona parámetros
   - main.py inyecta en LLM y RAG

3. Pipeline pattern
   - input → search → construct prompt → query → output

4. Factory pattern (simple)
   - config.get_model_path() → crea ruta dinámicamente

================================================================================
LÍMITES POR DISEÑO
================================================================================

Sin chat memory:
  ✗ No reconoce contexto de preguntas anteriores
  ✓ Cada pregunta es independiente
  ✓ Menor complejidad

Sin evaluadores:
  ✗ No mide calidad de respuesta
  ✓ Devuelve lo que el LLM genera
  ✓ Menor overhead

Sin caching:
  ✗ La misma pregunta se procesa 2 veces
  ✓ Simpler implementation
  ✓ RAM constante

Sin web UI:
  ✗ Solo CLI (stdin/stdout)
  ✓ Fácil integrar en scripts
  ✓ Sin dependencias web

Estos límites son INTENCIONALES.
Son trade-offs aceptados a cambio de simplificidad.
