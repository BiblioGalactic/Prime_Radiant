================================================================================
COMPONENTES EN DETALLE
================================================================================

1. CONFIG.PY (78 líneas)
   ├─ Detecta automáticamente rutas (modelo, índice, base)
   ├─ Parámetros de configuración (temp, tokens, contexto)
   ├─ Sin dependencias externas (solo pathlib, os)
   └─ Falla rápido si faltan archivos clave

2. LLM.PY (99 líneas)
   ├─ SimpleLLM: interfaz minimalista
   ├─ llama-cli como subprocess (sin daemons)
   ├─ query(prompt) → respuesta
   ├─ stream_query(prompt) → generador (tokens)
   └─ Manejo de timeouts y errores

3. RAG.PY (108 líneas)
   ├─ SimpleRAG: recuperación básica
   ├─ Carga índice FAISS + chunks.pkl
   ├─ Embeddings con sentence-transformers
   ├─ search(query) → contexto concatenado
   ├─ search_with_scores(query) → relevancia
   └─ K=5 fragmentos por defecto (configurable)

4. MAIN.PY (106 líneas)
   ├─ WikiRAGE5: clase controladora
   ├─ answer(question) → (respuesta, contexto)
   ├─ Flujo: pregunta → RAG search → LLM query → respuesta
   ├─ Loop interactivo simple (stdin/stdout)
   ├─ Opcional: mostrar contexto usado
   └─ Exit condition: "exit", "quit", "salir"

================================================================================
FLUJO DE EJECUCIÓN
================================================================================

python main.py
    │
    ├─→ Carga config.py (detecta rutas, parámetros)
    │
    ├─→ Inicializa SimpleLLM (verifica llama-cli)
    │
    ├─→ Inicializa SimpleRAG (carga FAISS + chunks)
    │
    └─→ Loop interactivo:
        │
        ├─→ Lee pregunta del usuario
        │
        ├─→ RAG.search(question)
        │   └─→ Embed query
        │   └─→ FAISS.search() → top-5 chunks
        │   └─→ Concatena contexto
        │
        ├─→ Construye prompt (context + question)
        │
        ├─→ LLM.query(prompt)
        │   └─→ Subprocess: llama-cli -m model.gguf -p prompt
        │   └─→ Lee stdout
        │   └─→ Retorna respuesta
        │
        └─→ Imprime respuesta + opción de ver contexto

================================================================================
CONFIGURACIÓN AUTOMÁTICA
================================================================================

Las rutas se detectan así:

BASE_PATH → get_base_path()
           └─→ Path(__file__).parent.parent.parent
               (3 niveles arriba de VERSIONE5/)

MODEL_PATH → get_model_path()
             └─→ BASE_PATH / models / *.gguf (primer .gguf)

INDEX_PATH → get_index_path()
             └─→ BASE_PATH / data / faiss_index / index.faiss

Todo es automático. Si faltan archivos, falla rápido con mensaje claro.

================================================================================
SOPORTE Y BUGS
================================================================================

Si algo no funciona:

1. ¿Error de config? → Revisa el mensaje de config.py
2. ¿No encuentra llama-cli? → Instala y agrega a PATH
3. ¿Index not found? → Ejecuta build_index.py
4. ¿Timeout? → Aumenta el timeout en llm.py
5. ¿OOM? → Usa modelo más pequeño

Todos los errores tienen mensajes claros.
